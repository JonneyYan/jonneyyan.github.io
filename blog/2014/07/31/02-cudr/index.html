插入（Insert）

insert()

  • 该方法会自动给文档增加一个_id键

batchInsert()

  • 只能一次插入到单个集合中
  • 中间有错误，后面的停止插入
  • 可以使用continueOnError选项忽略错误,shell不支持，驱动程序支持

删除（Delete）

删除所有文档, 但不删除集合

db.foo.remove()

删除单个文档

db.foo.remove({“opt-put” : true})

删除集合

db.foo.drop()

修改器（Modifier）

$set

  • 用来指定一个字段的值，如果这个字段不存在，则创建他。
  • 可以修改字段的类型，例如将一个字段改为数组
  • 使用$unset删除键
  • 可以修改内嵌文档

$inc

  • 用来增加或减少键的值，如果字段不存在，则创建它
  • 只能用于数值类型的值

数组修改器

  • $push 向数组末尾添加一个元素
    • $each 添加多个元素
    • $slice 保证数组不会超过设定好的最大长度
    • $sort 超过$slice指定的长度时，将元素排序，并保留前xx个元素
    • $sort 和 $slice需要配合$each使用
  • $ne && $addToSet 将数组当做数据集（set）操作
    • $addToSet 可以和$each配合使用
  • $pop && $pull 删除元素
    • $pop 将数组看成队列或者栈
- `$pull` 会将数组中所有匹配的文档删除, 例如`[1,2,3,1,1]`,执行`pull 1`结果为`[2,3]`
  • 基于位置的数组修改器
    • 使用数组的下标
    • 使用$定位操作符

修改器的速度

  • Mongodb 文档的存储方式：
    • 当文档被插入时，MongoDB会将文档依次存入硬盘 |“x” : “a”| |“x” : “b”| |“x” : “c”|
    • 在对中间的文档进行修改，使其尺寸变大，该文档就会被移动到集合的尾部.此时会修改集合的填充因子（padding factor）使其从1变为1.5 |“x” : “a”| |“x” : “c”| |“x” : “bbb”|
    • 所以，对于$inc和不发生文档变化的$set修改器，执行的速度是很快的

合并 || 迁移（upsert）

  • upsert 是一种特殊的更新，如果查找到符合的文档，就会正常更新，如果没有查找到符合的文档，那么就会以查询的文档和更新文档共同创建一个新的文档。这就相当于将查询条件看成是Master， 将更新文档合并到Master上。
- 原子操作
  • $setOnInsert 只在创建字段时赋值，之后字段的值不再改变
  • save (shell程序)
    • 如果文档不存在，创建文档并插入；
    • 如果文档存在，更新文档

更新多个文档

  • 默认情况下更新只对匹配到的第一个文件进行更新，如果想要更新多个文档，需要将update的第4个参数设置为true。
  • findAndModify 命令可以查找元素并返回更新后的文档
  • findAndModify 命令可以使用的字段：
    • query
    • sort
    • update
    • remove
    • new
    • fields
    • upsert
    • update 和 remove 必须有一个，且只能有一个。

写入安全（Write Concern）

  • 应答式写入(acknowledged write) 数据库给出响应
  • 非应答式写入(unacknowledged write)
